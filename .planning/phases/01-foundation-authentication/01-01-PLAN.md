     "compilerOptions": {

       "skipLibCheck": true,
       "strict": true,

<task type="auto">
  <name>Task 2: Set up local MongoDB Docker container</name>
  <files>.docker/mongodb/docker-compose.yml, .docker/mongodb/init-mongo.js, .gitignore</files>
  <action>
Create local MongoDB development environment:

1. Create directory structure: `.docker/mongodb/`
2. Create docker-compose.yml:
   ```yaml
   version: '3.8'
   services:
     mongodb:
       image: mongo:7.0
       container_name: focusflow-mongodb-dev
       ports:
         - "27017:27017"
       environment:
         MONGO_INITDB_DATABASE: focusflow
       volumes:
         - mongodb_data:/data/db
         - ./init-mongo.js:/docker-entrypoint-initdb.d/init-mongo.js:ro
       restart: unless-stopped

   volumes:
     mongodb_data:
   ```
3. Create init-mongo.js to initialize database with any default collections
4. Add .docker/ to .gitignore (ignore volume data, keep config files)
5. Add docker-compose commands to package.json scripts:
   ```json
   "docker:mongo": "docker-compose -f .docker/mongodb/docker-compose.yml up -d",
   "docker:mongo:stop": "docker-compose -f .docker/mongodb/docker-compose.yml down",
   "docker:mongo:logs": "docker-compose -f .docker/mongodb/docker-compose.yml logs -f mongodb"
   ```
6. Start MongoDB: `npm run docker:mongo`
7. Verify connection: `docker exec -it focusflow-mongodb-dev mongosh` (connects to mongo shell)

This implements D-10 (local Docker MongoDB) per CONTEXT.md decision.
  </action>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Next.js middleware | accept | No authentication yet — will be added in Plan 02 |
| T-01-02 | Tampering | Environment variables | mitigate | Use .env.local.example as template, never commit .env.local. Add to .gitignore. |
| T-01-03 | Repudiation | Audit logging | accept | No audit logging needed for project setup phase |
| T-01-04 | Information Disclosure | Error messages | mitigate | Next.js error pages show stack traces only in dev. In production, use generic error messages. |
| T-01-05 | Denial of Service | MongoDB connection pool | mitigate | Configure connection limits in docker-compose.yml (maxPoolSize: 10 in Plan 02) |
