## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | Untrusted task text crosses here (XSS risk) |
| client → server | Edit lock can be bypassed client-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | PATCH /api/tasks/[id] | mitigate | Server-side edit lock check: (Date.now() - submittedAt) > 5 minutes before allowing update |
| T-05-02 | Spoofing | POST /api/tasks/[roomId] | mitigate | Verify session.user.id matches task userId, prevent task creation for other users |
| T-05-03 | Tampering | Task text content | mitigate | Sanitize taskText, React's default XSS protection prevents script injection, no HTML rendering |
| T-05-04 | Information Disclosure | GET /api/tasks/[id] | mitigate | Verify userId matches session.user.id before returning task, prevent accessing others' tasks |
| T-05-05 | Tampering | task completion status | mitigate | Server-side validation via separate endpoint, store in SessionCompletion not Task model |
</threat_model>
