2. Add byte counters to MediaSoupRoom interface:
   ```typescript
   interface MediaSoupRoom {
     router: Router
     transports: Map<string, WebRtcTransport>
     producers: Map<string, Producer>
     consumers: Map<string, Consumer[]>
     bytesRelayed: Map<string, number>  // transportId -> bytes
     bytesDirect: Map<string, number>   // transportId -> bytes
   }
   ```

3. In `createWebRtcTransport`:
   - Initialize byte counters for the transport

4. Listen to transport 'tuple' event to detect connection type:
   - If tuple.protocol === 'udp' and tuple.remoteIP is public → direct
   - If tuple.protocol === 'tcp' or relay candidate → relay

5. Listen to transport 'close' event:
   - Get final byte counts from transport.getStats()
   - Call trackBandwidth with accumulated bytes

Alternative approach (simpler): Poll transport.getStats() every 30 seconds and aggregate, then close and record on transport close.

Note: Since server/ and src/ are separate packages, duplicate tracking logic in server/bandwidth-tracker.ts or create a shared package.
