Create `src/lib/logger.ts` with:

1. LogEntry interface:
   ```typescript
   interface LogEntry {
     timestamp: string  // ISO 8601
     level: LogLevel
     message: string
     context?: string  // Module or component name
     userId?: string
     roomId?: string
     error?: {
       name: string
       message: string
       stack?: string
     }
     metadata?: Record<string, any>
   }

   type LogLevel = 'debug' | 'info' | 'warn' | 'error'
   ```

2. Logger object with methods:
   - `debug(message, metadata)`
   - `info(message, metadata)`
   - `warn(message, metadata)`
   - `error(message, error, metadata)`
   - `setContext(context)` - Set default context for subsequent logs
   - `setUserId(userId)` - Set user context
   - `setRoomId(roomId)` - Set room context

3. Log formatting:
   - Development: Pretty-print to console with colors
   - Production: JSON output to stdout (structured for log aggregators)

4. Log level filtering:
   - Respect LOG_LEVEL env var (default: info in production, debug in development)
   - Filter out logs below configured level

5. Winston or pino integration: Use winston@3.x for production-grade logging with transports

Per D-14: Comprehensive server logging with structured JSON.
Per D-15: Different log levels with environment-based configuration.
