3. **In roomNamespace.on('connection'), add 'chat-message' handler:**
   ```typescript
   socket.on('chat-message', async (data) => {
     try {
       // Validate message with Zod
       const { message } = ChatMessageSchema.parse(data)

       // Rate limiting: check recent messages from this user
       const recentCount = await ChatMessage.countDocuments({
         userId: user.id,
         timestamp: { $gte: new Date(Date.now() - 60000) }
       })

       if (recentCount >= 10) {
         socket.emit('chat-error', { error: 'Rate limit exceeded. Please wait before sending another message.' })
         return
       }

       // Save to MongoDB FIRST
       const chatMessage = await ChatMessage.create({
         roomId: roomId,
         userId: user.id,
         userName: user.name,
         userPhoto: user.photoUrl,
         message: message
       })

       // Broadcast to room after successful save
       roomNamespace.to(roomId).emit('chat-message', {
         messageId: chatMessage._id.toString(),
         userId: user.id,
         userName: user.name,
         userPhoto: user.photoUrl,
         message: message,
         timestamp: chatMessage.timestamp.toISOString()
       })
     } catch (error) {
       if (error instanceof z.ZodError) {
         socket.emit('chat-error', { error: 'Invalid message format. Message must be 1-500 characters.' })
       } else {
         console.error('Error saving chat message:', error)
         socket.emit('chat-error', { error: 'Failed to send message. Please try again.' })
       }
     }
   })
   ```

4. **Add 'fetch-history' handler:**
   ```typescript
   socket.on('fetch-history', async (data) => {
     try {
       const { limit = 50 } = data || {}

       const recentMessages = await ChatMessage.find({
         roomId: roomId
       })
         .sort({ timestamp: -1 })
         .limit(limit)
         .lean()

       // Send in chronological order
       socket.emit('chat-history', {
         messages: recentMessages.reverse().map(msg => ({
           messageId: msg._id.toString(),
           userId: (msg.userId as any).toString(),
           userName: msg.userName,
           userPhoto: msg.userPhoto,
           message: msg.message,
           timestamp: msg.timestamp.toISOString()
         }))
       })
     } catch (error) {
       console.error('Error fetching chat history:', error)
       socket.emit('chat-error', { error: 'Failed to load message history' })
     }
   })
   ```

Follow RESEARCH.md Pattern 2: Save to MongoDB BEFORE broadcasting.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Chat message | Untrusted message content from authenticated users |
| Rate limiter → ChatMessage | Query to check message frequency |
| ChatMessage → Broadcast | Validated messages sent to room participants |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | Chat message content | mitigate | Zod schema validation (min 1, max 500 chars). Never trust client input. |
| T-03-07 | Denial of Service | Chat spam / message flood | mitigate | Rate limiting: max 10 messages/minute via countDocuments query. |
| T-03-08 | Tampering | Message injection | mitigate | Server-side validation before MongoDB save. Sanitize if adding rich text (future). |
| T-03-09 | Information Disclosure | Chat history leakage | mitigate | fetch-history only returns messages for user's current room. Namespace isolation enforced. |
| T-03-10 | Tampering | Phantom messages | mitigate | Save to MongoDB FIRST, then broadcast. Rollback on DB error. |
| T-03-11 | Denial of Service | MongoDB storage exhaustion | mitigate | Message max length 500 chars. Consider TTL index for old messages (RESEARCH.md Open Question #1). |
</threat_model>
