1. **Add Phase 3: Real-Time Infrastructure section:**

   ## Phase 3: Real-Time Infrastructure

   ### Overview
   Phase 3 implements real-time room presence and live text chat using Socket.IO, Zustand state management, and React hooks.

   ### Server Setup

   **Start Socket.IO server:**
   ```bash
   cd server
   node socket-server.ts
   ```
   Server runs on port 3001 (configurable via SOCKET_PORT env var).

   **Verify server running:**
   - Check console for "Socket.IO server running" message
   - Visit http://localhost:3001/socket.io/ (should show 200 OK)

   ### Manual Testing Procedures

   **Test 1: Presence Tracking**
   1. Start Next.js dev server: `npm run dev`
   2. Open browser to http://localhost:3000
   3. Join a test room (use existing room from Phase 2)
   4. Open second browser window to same room
   5. Verify ParticipantList shows both users
   6. Close first window, verify second shows "Participant left"
   7. Check browser console for heartbeat messages (every 15s)

   **Test 2: Live Text Chat**
   1. With two browser windows open to same room
   2. Type message in first window's ChatBox
   3. Click Send or press Enter
   4. Verify message appears in both windows
   5. Verify message shows correct timestamp
   6. Verify sender name displayed correctly
   7. Try sending empty message (should not send)
   8. Try sending message > 500 chars (should truncate)

   **Test 3: Chat History**
   1. Join room with existing messages
   2. Verify last 50 messages load automatically
   3. Messages should display in chronological order
   4. Scroll to bottom should work smoothly

   **Test 4: Rate Limiting**
   1. Send 10 messages quickly in one minute
   2. 11th message should show error toast
   3. Wait 1 minute, try again (should work)
   4. Check server logs for rate limit enforcement

   **Test 5: Reconnection Handling**
   1. Join room in browser
   2. Kill Socket.IO server (Ctrl+C)
   3. Wait for disconnect message in browser console
   4. Restart Socket.IO server
   5. Verify client auto-reconnects
   6. Verify participant list resyncs
   7. Verify chat history reloads
   8. Verify no duplicate messages

   **Test 6: Multiple Tabs**
   1. Open 3 tabs to same room
   2. Verify user counted once in participant list
   3. Close one tab
   4. Verify user still counted (2 tabs remaining)
   5. Close all tabs
   6. Verify user removed from presence

   ### Expected Behavior

   **Presence:**
   - Participant count updates immediately on join/leave
   - User photos display correctly (or fallback avatar)
   - No duplicate users (multiple tabs = one participant)
   - Heartbeat maintains presence (30s server timeout)

   **Chat:**
   - Messages appear in < 100ms across windows
   - Messages persist across page refresh
   - Oldest messages load first
   - Auto-scroll to bottom on new messages
   - Input disabled when disconnected

   **Reconnection:**
   - Exponential backoff (1s → 5s max)
   - 5 reconnection attempts before giving up
   - State resyncs automatically
   - No duplicate messages after reconnect

   ### Troubleshooting

   **Presence not updating:**
   - Check Socket.IO server running
   - Check browser console for connection errors
   - Verify JWT auth working (check Network tab)
   - Check server logs for heartbeat processing

   **Chat messages not appearing:**
   - Check server logs for chat-message events
   - Verify MongoDB connection (check server logs)
   - Check for rate limit errors in console
   - Verify room IDs match (client vs server)

   **Reconnection failing:**
   - Check Socket.IO server restarted successfully
   - Verify port 3001 not blocked by firewall
   - Check browser console for reconnection errors
   - Try manual page refresh

   ### Test Data

   **Create test room:**
   Use existing room from Phase 2 or create via admin API.

   **Create test users:**
   Use existing users from Phase 1 or register new accounts.
