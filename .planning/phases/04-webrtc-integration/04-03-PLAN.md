  <files>src/hooks/useWebRTCConnection.ts</files>
  <action>
    Create src/hooks/useWebRTCConnection.ts with mediasoup Device and transport management:

    1. **State management**:
       - device: Device | null
       - transport: any | null
       - producers: Map<string, Producer> (audio, video)
       - consumers: Consumer[]

    2. **Initialize Device on mount**:
       - Emit 'get-router-rtp-capabilities' to server
       - Receive router RTP capabilities
       - Call createDevice(rtpCapabilities)
       - Set device state

    3. **Create transport**:
       - Emit 'create-transport' request
       - Receive transport info (id, iceParameters, iceCandidates, dtlsParameters)
       - Create device.createSendTransport() or createRecvTransport()
       - Handle transport 'connect' event: emit 'connect-transport' with DTLS parameters
       - Handle transport 'produce' event: emit 'produce' with RTP parameters
       - Set transport state

    4. **Create producers** (after transport connected):
       - Call createProducer(transport, localStream, 'audio')
       - Call createProducer(transport, localStream, 'video')
       - Store in producers Map
       - Emit 'new-producer' to other participants via server

    5. **Listen for new producers**:
       - On 'new-producer' event, call createConsumer()
       - Add to consumers array
       - Return consumer object

    6. **Cleanup on unmount**:
       - Close all producers
       - Close all consumers
       - Close transport
       - Don't close device (singleton)

    Use the code example from RESEARCH.md §WebRTC Connection Hook (React).

    **DEPENDENCIES**: Requires useMediaStream hook for localStream.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → getUserMedia | User must grant camera/microphone permission |
| client → server signaling | Untrusted client sends transport/producer requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Spoofing | Media stream injection | mitigate | getUserMedia requires user permission, browser enforces security |
| T-04-12 | Tampering | Transport parameters | mitigate | Server validates all transport/producer parameters with Zod schema (plan 04-02) |
| T-04-13 | Information Disclosure | Local stream leakage | mitigate | Never store MediaStream in Zustand (not serializable), keep in hook state only |
| T-04-14 | Denial of Service | Producer exhaustion | mitigate | Limit to 2 producers per user (1 audio, 1 video), server enforces capacity |
| T-04-15 | Elevation of Privilege | Unauthorized stream access | accept | Client can only access own local stream, server enforces room isolation |
</threat_model>
