<task type="auto">
  <name>Task 3: Create room business logic with atomic operations</name>
  <files>src/lib/rooms.ts</files>
  <action>
Create src/lib/rooms.ts with:

1. **getTodaysRooms(): Promise<IRoom[]>**
   - const today = startOfDay(new Date())
   - const tomorrow = addDays(today, 1)
   - await Room.find({ scheduledTime: { $gte: today, $lt: tomorrow } }).sort('scheduledTime')
   - Return array of today's rooms

2. **registerForRoom(roomId: ObjectId, userId: ObjectId): Promise<IRoom>**
   - Check registration window (30 min before session)
   - Atomic operation: Room.findByIdAndUpdate(roomId, {
       $push: { participants: userId },
       $inc: { participantCount: 1 }
     }, { new: true, runValidators: true })
   - Add participantCount field to Room schema if missing
   - Throw error if room full

3. **cancelRegistration(roomId: ObjectId, userId: ObjectId): Promise<IRoom>**
   - Atomic operation: Room.findByIdAndUpdate(roomId, {
       $pull: { participants: userId },
       $inc: { participantCount: -1 }
     }, { new: true })

4. **updateRoomStatus(roomId: ObjectId, status: string): Promise<IRoom>**
   - Validate status enum
   - Update room.status
   - Handle status transitions (scheduled → open → in-progress → completed)

5. **getRoomParticipants(roomId: ObjectId): Promise<IUser[]>**
   - await Room.findById(roomId).populate('participants')
   - Return populated participants array

Follow RESEARCH.md Pattern 4: Use atomic operations to prevent race conditions.
