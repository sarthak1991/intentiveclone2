## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Socket.IO server | Untrusted WebSocket connections, must authenticate |
| Socket.IO server → NextAuth | Shared JWT secret for token verification |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Spoofing | Socket.IO authentication | mitigate | JWT verification in io.use() middleware using NEXTAUTH_SECRET. Reject unauthorized connections. |
| T-02-09 | Elevation of Privilege | Socket.IO user data | mitigate | Attach user to socket.data only after successful JWT verification and User.findById(). |
| T-02-10 | Information Disclosure | Socket.IO error messages | mitigate | Return generic "Authentication error" without revealing if user exists. |
| T-02-11 | Tampering | Signal events | mitigate | Validate event structure on server, don't trust client data. Forward only, don't execute. |
| T-02-12 | Denial of Service | Socket.IO connections | accept | Phase 2 only has admin users. Rate limiting in Phase 7. |
| T-02-13 | Spoofing | Room namespace access | mitigate | Namespace pattern validation (/room-{roomId}). Registration check in Phase 3. |
</threat_model>
